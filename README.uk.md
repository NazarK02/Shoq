# Shoq

[English](README.md) · [Português (Portugal)](README.pt-PT.md) · **Українська**

> Кросплатформний месенджер з наскрізним шифруванням, побудований на Flutter та Firebase.

---

## Що таке Shoq?

Shoq — це безпечний месенджер з відкритим кодом і наскрізним шифруванням (E2EE). Повідомлення шифруються на пристрої відправника й можуть бути розшифровані лише на пристрої отримувача. Firebase ніколи не отримує вміст у відкритому вигляді.

## Можливості

- **Наскрізне шифрування** — шифрування на кожен пристрій за допомогою X25519 + XChaCha20-Poly1305
- **Автентифікація** — електронна пошта/пароль та Google Sign-In
- **Особисті чати та сервери** — текстові, голосові, файлові канали, форум і завдання
- **Передача файлів та зображень** — з відображенням прогресу та платформо-специфічними директоріями
- **Голосові канали WebRTC** — вимкнення мікрофона, вимкнення звуку, перемикач динаміка, перевірка стану з'єднання
- **Push-сповіщення FCM** — без вмісту в payload (zero-content)
- **Присутність користувача** — з Firestore heartbeat
- **Виявлення контактів через QR** — за допомогою `qr_flutter` + `mobile_scanner`
- **Папки розмов** — власна організація
- **Вибір емодзі**, попередній перегляд посилань, статуси повідомлень

## Архітектура безпеки

### Що захищено
- **Вміст повідомлень** — шифрується на пристрої відправника до будь-якого запису у Firestore
- **Карта шифртексту на пристрій** — кожен пристрій отримувача отримує власну зашифровану копію
- **Приватні ключі** — ніколи не покидають пристрій, на якому були згенеровані; зберігаються через `flutter_secure_storage`

### Відомі обмеження (важливо повідомляти користувачів)
- **Відсутня верифікація ключів.** Немає порівняння відбитків ключів чи механізму номерів безпеки. Зловмисник із доступом до Firestore може підмінити публічний ключ (атака TOFU / підміна ключа). Це найсерйозніша відкрита проблема безпеки.
- **Метадані не захищені.** Firebase спостерігає, хто з ким спілкується, коли і як часто. Шифрується лише вміст повідомлень.
- **Ротація ключів не реалізована.** Заплановано на Фазу 06. Forward secrecy не є повним до реалізації ротації.

## Швидкий старт

```bash
# Необхідно: Flutter SDK, Firebase CLI, обліковий запис Firebase

git clone https://github.com/NazarK02/Shoq.git
cd Shoq

flutter pub get
flutter gen-l10n

# Налаштуйте ваш проєкт Firebase та додайте google-services.json / GoogleService-Info.plist
flutter run
```

## Попередження щодо безпеки

Цей застосунок перебуває в активній розробці. Описана вище відсутність верифікації ключів є реальним вектором атаки. Не використовуйте його в контекстах, де секретність від зловмисника з доступом до бази даних є критичною вимогою — до реалізації верифікації відбитків ключів (Фаза 06).
